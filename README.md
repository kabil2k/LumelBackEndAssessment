
# Flash Sale Backend Assessment
 
## Sections Covered
1. SQL - Reliable Drivers
2. API - Flash Sale Inventory System
3. Algorithm - Server Scheduler
 
---
 
## API Highlights
- Thread-safe reservation using locks
- Waitlist using FIFO queue
- Reservation expiry via background service
 
---
 
## SQL Highlights
- Aggregation with HAVING clause
- Success rate calculation using conditional SUM
 
---
 
## Algorithm Approach
- Greedy (sort by end time)
- Time Complexity: O(N log N)
 
---
 
## Improvements
- Use Redis for distributed locking
- Add rate limiting
- Persist data in DB instead of memory
